# 농구 뉴스 자동 메일 🏀

Google 뉴스 RSS로 농구 관련 기사를 매일 모아 이메일로 보내주는 도구입니다.
(위험관리 EWI 모니터와 같은 구조 — Node.js + nodemailer + Gmail + 작업 스케줄러)

## 동작
1. `.env`의 `NEWS_KEYWORDS` 각 키워드로 Google 뉴스를 검색
2. 최근 `NEWS_HOURS`시간(기본 24h) 기사만 추림 (없으면 최신 기사로 대체)
3. 키워드 간 중복 기사 제거
4. **이미 보낸 기사 제외** — 지난 발송 이후 새로 뜬 기사만 (이력은 `data/seen.json`)
5. 각 기사 원문에서 요약(og:description) 추출 (실패 시 제목만)
6. HTML 이메일로 정리해 `NEWS_MAIL_TO`로 발송. 새 기사가 없으면 발송하지 않음.

### '용산 소식' 챕터(메일 하단)
- `NEWS_YONGSAN_KEYWORDS`의 키워드(용산중·용산고·신석 감독·이정석 코치)는 매일 수집돼, 메일 **맨 아래 '🏫 용산 소식' 챕터**에 한데 모입니다. 기사가 있을 때만 표시됩니다.
- 매일(주말 포함) 08:30 발송에 함께 나갑니다.

### 주간 키워드(일부 키워드는 1주일에 한 번만)
- `NEWS_WEEKLY_KEYWORDS`에 넣은 키워드(예: `용산중 농구,용산고 농구`)는 매일이 아니라 **`NEWS_WEEKLY_DAYS`(기본 7)일에 한 번만** 포함됩니다.
- 포함되는 날엔 그 키워드의 **지난 7일치**를 한 번에 담고, 이후 7일간은 건너뜁니다(상태: `data/weekly.json`).
- 나머지 키워드는 평소대로 매일. 수집 실패(네트워크)로 발송을 못 한 날은 주기를 소진하지 않아 다음날 재시도합니다.

### 증분 발송(한 번 보낸 기사는 다시 안 보냄)
- 발송에 성공한 기사 제목은 `data/seen.json`에 기록되고, 다음 발송 때 제외됩니다(30일 후 자동 정리).
- **처음 시작할 때** 현재 떠 있는 기사를 "이미 본 것"으로 표시해 두려면: `npm run seed`
  → 이후 발송부터는 그 시점 이후 새로 나온 기사만 나갑니다.

## 설정
`.env` 파일에서 조정:
- `NEWS_KEYWORDS` — 검색어(콤마구분). 예: `농구,KBL,WKBL,NBA,유소년 농구`
- `NEWS_MAIL_TO` — 받는 사람(콤마로 여러 명)
- `NEWS_HOURS` — 최근 몇 시간 기사만 (기본 24)
- `NEWS_MAX_PER_KEYWORD` — 키워드별 최대 개수 (기본 6)

Gmail은 **앱 비밀번호**가 필요합니다(일반 비번 아님). 2단계 인증 → 앱 비밀번호 발급 → `SMTP_PASS`.

## 사용
```
npm install          # 최초 1회 (nodemailer 설치)
npm run seed         # (선택) 현재 기사를 '이미 본 것'으로 표시 — 시작 시 1회
npm run preview      # 발송 없이 수집 결과 확인 + data/preview.html 저장
npm run send         # 실제 메일 발송 1회(새 기사 있을 때만)
```

## 매일 자동 발송 (Windows 작업 스케줄러)
1. `taskschd.msc` 실행 → 작업 만들기
2. 트리거: 매일 원하는 시각(예: 08:00)
3. 동작: 프로그램 시작 → `cmd.exe`, 인수 `/c run-daily.bat`, 시작 위치 = 이 폴더
4. 로그는 `data/send.log`에 남습니다.

> ⚠️ `.bat`을 직접 지정하면 실행이 안 되는 경우가 있어 `cmd.exe /c run-daily.bat`로 감싸는 것을 권장합니다.
